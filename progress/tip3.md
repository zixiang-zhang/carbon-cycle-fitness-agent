# 一、核心定位（一句话）

> 一个能**根据目标、反馈和执行情况，动态调整高碳 / 中碳 / 低碳饮食策略的智能健康 Agent**

⚠️ 明确声明：**非医疗建议，仅用于生活方式管理**

---

# 二、基础功能（必须做，保证可用）

这些功能保证它是一个“完整产品”，而不是玩具。

## 1️⃣ 用户目标与画像建模

**Agent 要先“认识你”**

* 目标：减脂 / 维持 / 增肌
* 基本信息：身高、体重、性别、活动水平（低/中/高）
* 饮食偏好：外卖 / 自炊、忌口、口味偏好
* 可坚持度：严格 / 灵活

👉 **Agent 价值点**：
不是表单，而是**对话式收集 + 状态存储（Memory）**

---

## 2️⃣ 碳循环计划自动生成（Plan）

* 自动生成 **7 天碳循环安排**

  * 高碳日 / 中碳日 / 低碳日
* 结合：

  * 训练日 vs 非训练日
  * 工作日 vs 周末
* 输出内容：

  * 每日类型（HC / MC / LC）
  * 简单饮食结构（不用算卡路里到克）

📌 示例输出（非常 demo 友好）：

> 周一：高碳（力量训练）
> 周二：低碳（休息）
> 周三：中碳（有氧）

---

## 3️⃣ 每日饮食记录（Act）

* 支持：

  * 文本记录（“中午吃了米饭+鸡胸”）
  * 简单标签（外卖 / 自炊）
* Agent 自动：

  * 粗略判断当天偏高 / 中 / 低碳
  * 不追求精准，追求可坚持

👉 这是“Agent 能判断现实执行情况”的关键

---

# 三、Agent 特色功能（拉开和普通 App 的差距）

这些是**面试官最爱问的部分**。

## 4️⃣ 执行偏差分析 Agent（Reflect）

**这是整个项目的灵魂**

Agent 每天 / 每周自动做：

* 计划 vs 实际 对比
* 偏差分析：

  * 连续高碳？
  * 低碳日坚持失败？
* 归因：

  * 社交？
  * 工作压力？
  * 计划不合理？

📌 示例输出：

> 你在低碳日连续 2 次外卖，可能与你最近加班有关。

---

## 5️⃣ 策略动态调整（Adjust）

Agent **不会死板执行原计划**：

* 连续失败 → 降低难度
* 连续成功 → 微调挑战
* 特殊事件（聚餐 / 出差）→ 临时重排碳循环

📌 示例：

> 本周低碳日成功率只有 40%，我已将下周低碳日从 3 天调整为 2 天。

👉 **这是 Agent，不是日历**

---

## 6️⃣ 训练日 / 生活节奏联动

* 用户标注：

  * 今天是否训练
  * 训练类型（力量 / 有氧）
* Agent 自动：

  * 推荐高碳日靠近训练日
  * 给出“碳补偿建议”

---

# 四、进阶 Agent 功能（技术深度体现）

做 1–2 个就已经非常能打。

## 7️⃣ 多 Agent 分工架构（强烈推荐）

| Agent         | 职责       |
| ------------- | -------- |
| Planner Agent | 生成碳循环方案  |
| Logger Agent  | 解析用户饮食描述 |
| Analyst Agent | 分析执行偏差   |
| Coach Agent   | 给调整建议    |

📌 面试时你可以直接画架构图讲。

---

## 8️⃣ 长期趋势分析 & 周报生成

* 体重变化（模拟）
* 碳循环执行率
* 哪种日型最容易失败
* 自动生成周总结报告（文字 + 图表）

---

## 9️⃣ 知识增强（RAG）

* 内置：

  * 什么是碳循环
  * 高碳 / 低碳食物示例
* Agent 回答问题时：

  * **结合用户当前阶段解释**
  * 引用来源，避免胡说
