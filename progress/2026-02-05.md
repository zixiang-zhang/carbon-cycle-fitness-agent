## 1. 后端 API 修复

### 问题现象
- 前端页面卡在登录窗口
- API 请求无响应（curl 超时）

### 问题原因
- 多个 Python 进程同时监听 8000 端口
- uvicorn 热重载未正确加载新代码

### 解决方案
```powershell
# 杀掉所有占用 8000 端口的进程
Get-NetTCPConnection -LocalPort 8000 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }

# 清除 Python 缓存
Remove-Item -Recurse -Force app/__pycache__, app/api/__pycache__

# 重启后端
python run_api.py
```

### 验证结果
- `/docs` 返回 200 ✅
- `/api/auth/login` 正常工作 ✅

## 3. 饮食记录 - 手动输入功能

### 需求描述
除了拍照识别，还需要支持手动输入食物名称和重量，系统自动估算营养成分（也可手动输入）。

### 后端实现

#### 新增 API 端点 (`app/api/food.py`)

```python
class ManualFoodInput(BaseModel):
    food_name: str      # 食物名称
    weight_g: float     # 重量（克）
    user_id: str
    meal_type: str = "lunch"
    auto_log: bool = True

@router.post("/estimate", response_model=FoodAnalysisResult)
async def estimate_food_nutrition(input_data: ManualFoodInput):
    """使用 LLM 根据食物名称和重量估算营养成分"""
    
    prompt = f"""作为营养师，请根据以下信息估算食物的宏量营养素：
    食物名称：{input_data.food_name}
    重量：{input_data.weight_g}克
    
    请以JSON格式返回：
    {{"carbs_g": <碳水>, "protein_g": <蛋白质>, "fat_g": <脂肪>}}
    """
    
    response = await llm.chat([{"role": "user", "content": prompt}])
    # 解析 JSON 并返回结果
```

#### API 测试结果
```bash

}

  "carbs_g": 44.0,
  "protein_g": 5.0,

  "calories": 199.6,

```

### 前端实现 (`frontend/src/components/FoodUploadModal.tsx`)

完全重写了组件，支持两种输入模式：

#### 模式切换

```tsx
    <button onClick={() => setInputMode("manual")}>✏️ 手动输入</button>

```

#### 手动输入表单
- 食物名称输入框
- 重量输入框（默认 100g）
- AI 自动估算开关
  - 开启：调用 `/api/food/estimate`
  - 关闭：直接进入营养数值编辑模式

#### 用户流程
```
1. 点击"记录饮食"卡片
2. 选择"✏️ 手动输入"标签
3. 输入食物名称（如"鸡胸肉"）和重量（如"150g"）
4. 开启 AI 估算 → 点击"🤖 AI 估算营养"
5. 等待 LLM 返回结果（约 2-3 秒）
6. 查看估算结果，可点击"手动修正数值"调整
7. 确认记录
```

## 4. 周复盘报告功能

### 功能概述
用户可在仪表盘点击"周复盘报告"卡片，进入报告页面，生成 AI 驱动的深度周报分析。

### 后端实现 (`app/api/report.py`)

#### API 端点
```python
@router.post("/weekly", response_model=WeeklyReportResponse)
async def generate_weekly_report(request: WeeklyReportRequest):
    """使用 Agent 生成 AI 周报"""
```

#### 报告生成流程
1. 获取用户本周计划数据和实际执行数据
2. 计算统计指标（热量达成率、训练完成率、宏营养素摄入等）
3. 调用 Agent 进行深度分析
4. 生成结构化报告，包含：
   - 📊 执行情况反馈
   - 📈 趋势分析
   - 💡 针对性改进建议
   - 🎯 Coach 寄语

#### Agent 调用
```python
agent = CarbonCycleAgent(memory_store=get_memory_store())

result = await agent.run(
    user_id=request.user_id,
    intent="reflect",
    trigger=f"weekly_report: {trigger_message}",
)
```

### 前端实现 (`frontend/src/app/report/page.tsx`)

#### 功能特性
- 本周数据概览（热量、训练、宏营养素）
- AI 深度分析报告生成

## 技术笔记

### Windows 端口占用排查
```powershell
# 查看端口占用
netstat -ano | findstr :8000

# 按进程 ID 杀掉
taskkill /PID <pid> /F

# 或使用 PowerShell
Get-NetTCPConnection -LocalPort 8000 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```

### uvicorn 热重载问题
- 修改 `app/api/*.py` 后可能不会自动重载
- 解决：手动重启或清除 `__pycache__`

### LLM 营养估算 Prompt 设计
- 要求返回 JSON 格式便于解析
- 设置 fallback 值防止解析失败
- confidence 设为 0.7（低于图片识别的置信度）
