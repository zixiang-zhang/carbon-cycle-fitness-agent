# 2026-01-31 Progress Report

## 📅 今日主要进展 (Key Achievements)

今日主要工作集中在 **RAG (检索增强生成) 基础设施的建设与稳定化**。经历了从 ChromaDB 到 Qdrant 的技术选型变更，最终解决了 Windows 环境下的兼容性问题，并完成了 RAG 模块的代码重构。

### 1. 🏗️ RAG 基础设施迁移与实现
*   **废弃 ChromaDB**: 由于 `chromadb` 在 Windows 环境下存在严重的依赖冲突（`onnxruntime`, `hnswlib` DLL 问题及 `numpy` 2.0 兼容性问题），决定将其移除。
*   **采用 Qdrant (Local Mode)**: 迁移至 `qdrant-client` (v1.9.1)，使用其本地文件模式（SQLite 后端）。
    *   **优势**: 无需 Docker，纯 Python/Rust 依赖，在 Windows 上运行极其稳定，且生产环境可无缝切换至 Server 版。
*   **ID 生成逻辑优化**: 修复了 Qdrant 对 ID 格式的严格要求，实现了自动 UUID 生成与原始 ID 的 Metadata 映射。

### 2. 🧠 混合检索系统 (Hybrid RAG)
*   **HybridRetriever**: 实现了结合 **语义检索 (Vector)** 和 **关键词检索 (BM25)** 的混合检索器。
    *   **向量权重**: 0.7 (侧重语义理解)
    *   **BM25 权重**: 0.3 (侧重专有名词精确匹配)
*   **文档处理**: 集成 `unstructured` 库，实现了对 Markdown 文档的智能标题切分（Chunk by Title），大幅提升了上下文语义的完整性。

### 3. ♻️ 代码重构与清理 (Refactoring)
*   **模块独立化**: 创建了独立的 `app/rag/` 模块，将 RAG 相关组件从 `app/llm/` 中分离，职责更清晰。
    *   `app/rag/retriever.py`: 文档加载与混合检索逻辑。
    *   `app/rag/vectorstore.py`: Qdrant 数据库交互封装。
    *   `app/rag/embedding.py`: 嵌入模型服务。
*   **配置优化**: 将 `app/core/config.py` 中特定的 `chroma_` 配置项重命名为通用的 `vector_store_` 配置项。
*   **清理**: 移除了所有 ChromaDB 相关数据目录 (`data/chroma`) 和临时测试脚本。

### 4. ✅ 全面服务验证
更新并通过了 `tests/test_services.py`，确认所有外部服务正常工作：
*   **LLM API**: 阿里云百炼 (Qwen-Plus) 连接正常。
*   **Embedding**: Ollama (bge-m3) 向量生成正常。
*   **VectorStore**: Qdrant 文档存取与检索正常。
