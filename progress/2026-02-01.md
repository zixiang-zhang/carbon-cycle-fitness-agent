## 🚀 后端核心成就 (Backend Achievements)

我们成功构建了一个基于 **LangChain/LangGraph** 的复杂 Agent 系统，实现了从用户画像到智能决策的完整闭环。

### 1. 核心架构 (8/8 模块全部通过)
- **Core (基础层)**: 实现了基于 Pydantic 的配置管理 (`Config`) 和异步数据库引擎 (`DatabaseEngine`)，确保系统的类型安全和并发性能。
- **Models (数据模型)**: 
  - 定义了 **Domain-Driven Design (DDD)** 风格的模型：`UserProfile` (TDEE自动计算), `CarbonCyclePlan` (周期逻辑), `DietLog` (饮食记录).
  - 实现了 `MacroNutrients` 值对象，自动计算热量分布。
- **Services (业务逻辑)**: 
  - `CarbonStrategyService`: 封装了高/中/低碳的核心算法。
  - `KnowledgeService`: 集成了 RAG 检索，支持查询食物营养和训练知识。

### 2. 智能体系统 (Agent System)
- **LangGraph 工作流**: 
  - 设计了 **Planner → Executor → Reflector** 的图结构。
  - 实现了 **State Management**，支持多轮对话上下文保持。
- **Memory (记忆系统)**: 
  - `AgentMemory`: 基于 SQLite 的持久化记忆，记录每一次 `run_id` 的决策路径 (`Chain of Thought`).
  - `UserMemory`: 长期记忆用户的偏好和历史反馈。
- **Prompt Engineering**: 
  - 优化了 Planner 的 Prompt，能够准确识别用户意图（如“我今天好累”）并调整碳水策略。

### 3. 数据持久化 (Persistence)
- 放弃了纯内存存储，升级为 **SQLite + SQLAlchemy (Async)**。
- 实现了 Repository 模式 (`UserRepository`, `PlanRepository`)，解耦了业务逻辑和数据库操作。
- 完成了完整的 CRUD 测试，确保数据在服务重启后不丢失。

## 🛠️ 遇到的挑战与解决方案 (Challenges & Solutions)

### 后端难点

1. **LangGraph 状态持久化**
   - **问题**: Agent 运行结束后状态丢失，无法进行多轮对话。
   - **解决**: 引入 `AgentMemory` 模块，将每一步的 `AgentState` 序列化存储到 SQLite `agent_runs` 表中，支持断点恢复。

2. **异步数据库测试**
   - **问题**: `aiosqlite` 在 pytest 中经常报错 `Task attached to a different loop`。
   - **解决**: 使用 `pytest-asyncio` 插件，并重写 `conftest.py` 中的 `event_loop` fixture，确保所有测试共享同一个事件循环。

3. **Pydantic 模型循环引用**
   - **问题**: `User` 和 `Plan` 互相引用导致序列化死循环。
   - **解决**: 拆分 Read/Write 模型 (`PlanCreate` vs `PlanResponse`)，并在 Repository 层手动处理关联 id。

### 前端难点

4. **TSX 中文解析错误**
   - **问题**: 在 Next.js 页面中直接使用中文双引号导致编译失败。
   - **解决**: 所有的 Mock 数据文本改用标准转义或避免在 JSX 属性外直接嵌套复杂引号。

5. **API 路由 404**
   - **问题**: Next.js 代理请求到 `/api/users`，但 FastAPI 默认挂载在根路径 `/users`。
   - **解决**: 在 FastAPI `main.py` 中使用 `app.include_router(api_router, prefix="/api")` 标准化所有路由。
